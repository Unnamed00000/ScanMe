# ScanMe

Сервис для цифровых визиток и объявлений. Администратор создаёт публикацию, а ScanMe сразу выдаёт постоянную ссылку и QR-код. После сканирования открывается отдельная полноэкранная страница выбранной визитки или объявления.

## Возможности

- два формата публикаций: «Визитка» и «Объявление»;
- любое количество публикаций в защищённой админке;
- автоматический адрес и QR-код, который не меняется после редактирования;
- 17 оформлений: цвета, автомобили, чёрный Land Rover Discovery 2019, лев, волк, орёл, горы, лес и времена года;
- отдельное романтическое оформление;
- для объявлений: заголовок, категория, цена, описание, фотография, автор, срок действия и кнопка связи;
- телефон, email, сайт, Telegram, WhatsApp и адрес;
- сохранение контакта в формате vCard;
- черновики и опубликованные материалы;
- мобильная полноэкранная версия для переходов по QR-коду.

## Локальный запуск

```bash
npm install
npm run dev
```

Без файла `.env` приложение запускается в демонстрационном режиме и хранит публикации только в `localStorage` текущего браузера. Такой QR-код нельзя использовать на других устройствах.

## Firebase

Приложение подключено к проекту `scanme-da22f`. В Firebase должны быть включены Email/Password в Authentication, создан пользователь-администратор и база Firestore. Домен `unnamed00000.github.io` должен находиться в списке Authorized domains.

Правила базы публикуются командой:

```bash
npx firebase-tools deploy --only firestore:rules
```

Опубликованные материалы доступны всем по прямой ссылке, а список и редактирование — только авторизованному администратору.

## GitHub Pages

В **Settings → Pages** используется **Deploy from a branch**, ветка `main`, папка `/ (root)`. Команда `npm run build` обновляет production-файлы в корне репозитория, после чего их нужно отправить в `main`.

Публичная ссылка выглядит так:

```text
https://USERNAME.github.io/ScanMe/#/p/адрес-публикации
```
